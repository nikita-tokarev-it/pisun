# Структура базы данных (Supabase PostgreSQL)

В данном документе описана структура базы данных, которая используется проектом. База данных спроектирована для работы с Supabase REST API (поверх PostgreSQL).

## Общая архитектура

Приложение использует 9 основных таблиц. 
Большинство таблиц работают независимо для хранения контента, но таблицы советов (`councils`, `regional_councils`) обладают встроенной логикой сортировки.

---

## 1. Таблица: `users`
Хранит учетные записи для доступа к административной панели (CMS).

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `username` | `text` (Unique) | Логин для входа |
| `password` | `text` | Хэшированный пароль (используется bcrypt/обертка) |
| `role` | `text` | Роль пользователя (`admin` или `editor`) |
| `created_at` | `timestamp` | Дата создания |

---

## 2. Таблица: `settings`
Хранит глобальные настройки сайта (JSON-структура). Приложение ожидает ровно одну запись (где `id = 1`).

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `integer` (Primary Key) | Всегда равен 1 |
| `data` | `jsonb` | Весь конфигурационный объект (header, footer, contacts, структура 'О совете') |
| `updated_at` | `timestamp` | Дата последнего изменения через CMS |

---

## 3. Таблица: `councils`
Хранит данные о составе Президиума (Главный Совет).

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `council` | `text` | Название совета/группы (для группировки) |
| `university` | `text` | Название университета |
| `chairman` | `text` | ФИО ректора/руководителя |
| `secretary` | `text` | ФИО секретаря (опционально) |
| `website` | `text` | Ссылка на сайт |
| `position` | `integer` | Порядковый номер внутри группы |
| `group_position`| `integer` | Порядковый номер самой группы (для сортировки групп целиком) |
| `created_at` | `timestamp` | Дата добавления |

---

## 4. Таблица: `regional_councils`
Хранит данные о Региональных советах (по субъектам РФ).

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `region` | `text` | Название региона (для группировки) |
| `organization`| `text` | Название организации/вуза |
| `chairman` | `text` | ФИО руководителя |
| `position_name`| `text` | Текстовая должность (например, "И.о. ректора") |
| `phone` | `text` | Контактный телефон |
| `email` | `text` | Электронная почта |
| `website` | `text` | Ссылка на сайт регионального отделения |
| `position` | `integer` | Порядковый номер внутри региона |
| `group_position`| `integer` | Порядковый номер самого региона |

---

## 5. Таблица: `events`
Мероприятия и Новости сайта (публикуются на главной странице и в разделе "Мероприятия").

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `title` | `text` | Заголовок новости |
| `date` | `date` | Дата публикации/проведения |
| `description` | `text` | Краткое описание (анонс) |
| `content` | `text` | Полный HTML/текстовый контент |
| `image` | `text` | URL обложки (по умолчанию: `/image.png`) |
| `created_at` | `timestamp` | - |

---

## 6. Таблицы для прессы: `press_releases`, `announcements`
Структура этих двух таблиц идентична. Используются для публикации пресс-релизов и анонсов для СМИ.

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `title` | `text` | Заголовок |
| `date` | `date` | Дата релиза |
| `content` | `text` | Содержание |

---

## 7. Медиа-материалы: `photos`, `videos`
Структура идентична. Используются для раздела "Для прессы" -> Медиа-материалы.

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `title` | `text` | Подпись к фото/видео |
| `url` | `text` | Прямая ссылка на файл (jpg) или видео-плеер/iframe |
| `date` | `date` | Дата добавления/события |

---

## 8. Таблица: `documents`
Раздел "Документы" (нормативные акты, положения).

| Колонка | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `uuid` (Primary Key) | Уникальный идентификатор |
| `title` | `text` | Название документа |
| `date` | `date` | Дата принятия/публикации документа |
| `url` | `text` | Ссылка на файл (PDF/DOCX) для скачивания |

---

## Особенности связей (Relations)
В текущей архитектуре базы данных **отсутствуют жесткие внешние ключи (Foreign Keys)** между таблицами (например, `events` не привязаны жестко к `users`), так как приложение работает как публичная доска объявлений, управляемая администраторами через CMS.

Главная связующая логика реализована на уровне сортировки: поля `group_position` и `position` в таблицах советов определяют иерархию отрисовки данных на Frontend-е.
