# Подробная структура проекта DVFU-Rectorat

Этот документ содержит детальное описание каждого файла и директории проекта, а также описывает основные процессы взаимодействия (workflows).

---

## 1. Общая архитектура
Проект представляет собой **Fullstack SPA** (Single Page Application).
- **Frontend**: React приложение.
- **Backend**: Двойная реализация. 
    - `server/` — Express сервер для локальной разработки.
    - `api/` — Serverless функции для деплоя на Vercel.
- **Data**: Имитация БД через JSON-файлы или in-memory объекты.

---

## 2. Детальное описание директорий и файлов

### 📁 Корневая директория
- `package.json` — Основные зависимости и скрипты запуска.
- `vercel.json` — Конфигурация деплоя, роутинга и проксирования для Vercel.
- `PROJECT_STRUCTURE.md` — Данное руководство.
- `INFO.md` — Дополнительная информация о проекте.

### 📁 Frontend (`src/`)
Основа пользовательского интерфейса.

#### `src/api/` (Клиентские запросы)
- `client.js` — Настройка **Axios**. Определяет базовый URL (localhost или `/api`) в зависимости от окружения. Добавляет JWT токен в заголовки.
- `auth.js`, `events.js`, `press.js` и др. — Модули с функциями для вызова конкретных эндпоинтов API.

#### `src/components/` (Компоненты)
- `Header/`, `Footer/` — Основные элементы навигации.
- `Layout/` — Базовая обертка страниц.
- `AdminLayout/` — Специальная обертка для админ-панели с боковым меню.
- `ProtectedRoute/` — Компонент-фильтр, закрывающий доступ к страницам для неавторизованных пользователей.
- `RussiaMap/` — Сложный компонент интерактивной карты (Paths, Regions, Data).

#### `src/context/`
- `AuthContext.js` — Глобальное состояние авторизации. Хранит данные пользователя и методы `login/logout`.

#### `src/pages/` (Страницы)
- `Home/`, `About/`, `Contacts/` — Информационные страницы.
- `Events/`, `Documents/`, `Press/` — Страницы с динамическим контентом (списки и детальные виды).
- `Login/` — Форма входа в систему.
- **Admin/** — Поддиректория с менеджерами для CRUD операций (создание, редактирование, удаление).

---

### 📁 Backend: Serverless API (`api/`)
Эти файлы исполняются в среде Vercel как независимые функции.
- `api/*.js` — Обработка публичных GET запросов.
- `api/auth/login.js` — Логика аутентификации, генерация JWT.
- `api/admin/*.js` — Защищенные функции. Проверяют токен и роль перед выполнением действий (POST/PUT/DELETE).

#### `lib/` (Общая логика API)
- `db.js` — Реализация доступа к данным. В Serverless среде использует in-memory объект.
- `auth.js` — Middleware для проверки JWT токена.

---

### 📁 Backend: Local Server (`server/`)
Используется только для разработки.
- `index.js` — Главный файл Express. Подключает маршруты и запускает сервер на порту 5001.
- `db.json` — Локальное хранилище данных. Изменения сохраняются здесь перманентно.
- `routes/` — Локальные обработчики, повторяющие логику Serverless функций.
- `utils/db.js` — Утилита для чтения/записи в `db.json`.

---

### 📁 Дополнительные модули
- `russia-map/` — Изолированный статический модуль с SVG картой (чистый HTML/JS), который может использоваться как iframe или внедряться в React.
- `public/` — Статика: `index.html` (каркас SPA), иконки, манифест.

## 3. Workflow (Рабочие процессы)

### Процесс разработки (Development)
1. Разработчик запускает `npm run dev`.
2. Команда `concurrently` запускает одновременно:
    - **React Dev Server** (порт 3000).
    - **Express Server** (порт 5001).
3. При запросе данных Frontend обращается к `localhost:5001/api/...`.
4. Изменения в админке записываются в `server/db.json`.

### Процесс авторизации (Auth Workflow)
1. Пользователь вводит логин/пароль на странице `/login`.
2. `AuthContext` вызывает `api/auth/login.js`.
3. Сервер проверяет данные, создает JWT и возвращает его клиенту.
4. Браузер сохраняет токен в `localStorage`.
5. При каждом следующем запросе `client.js` добавляет токен в заголовок `Authorization: Bearer <token>`.

### Взаимодействие с данными (Data Flow)
1. **Запрос**: Пользователь заходит на "События" -> Срабатывает `useEffect` в `Events.js`.
2. **Fetch**: Вызывается `src/api/events.js` -> `axios.get('/api/events')`.
3. **Server**: Маршрут принимает запрос, читает данные через `readDb()` (из файла или объекта).
4. **Response**: Сервер отдает JSON.
5. **UI**: React сохраняет данные в `useState` и отрисовывает список.

### Деплой (Deployment)
1. Проект пушится в Git репозиторий, связанный с Vercel.
2. Vercel собирает Frontend (`npm run build`).
3. Папка `api/` автоматически превращается в набор эндпоинтов.
4. Правила из `vercel.json` обеспечивают корректную работу SPA роутинга (перенаправляют все не-API запросы на `index.html`).

## 4. Ключевые скрипты
- `npm start` — Запуск только фронтенда.
- `npm run server` — Запуск только локального бэкенда.
- `npm run dev` — Полноценная среда разработки (Front + Back).
- `npm run seed` (в папке server) — Сброс и заполнение базы тестовыми данными.
